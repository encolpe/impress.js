Docker avec Django

Utilisation de conteneur pour le développement Python

Encolpe DEGOUTE

  • Open Source : 1996
  • Python : 2002
  • Framework/CMS : CPS, Plone, Django, Flask
  • Principales contributions : Workflows, Template de développement

Postulats
  • Le développement est le reflet de la production
  • Le développe choisi son environnement de travail
  • Le passage du développement vers la production doit être fluide
  • L'outil de construction doit laisser la plus petite empreinte possible dans le code du projet
Contraintes
  • Doit pouvoir contenir plus que du Python (Postgres, Redis, MQ, Logstash, ElasticSearch)
  • Facile à déployer, automatision des tests
  • Linux/MacOS/Windows
  • Un non développeur doit pouvoir l'installer en une ligne de code
Pourquoi Docker
  • Déjà testé : zc.buildout, lxc, VirtualBox
  • Les plus : projet très actif, intéresse les diplomés, très suivi par des gros contributeurs
  • Les moins : victime d'incompatibilité chronique, business model incertain (futur SCO ?)
Docker Engine
  • Un seul fichier DockerFile en YAML
  • Possibilité de démarrer avec un OS déjà pré-configuré
  • Possibilité de partir avec un OS vide, d'installer des dépendances et de spéciliser une configuration
  • Possibilité de sauver une image déjà construite prête à l'emploi
Docker Compose
  • Construction de plusieurs conteneurs au sein d'une unité
  • Possibilité de liéer profondémment certains services
  • Partage de paramètre
Orchestration
  • Couche propriétaire pour le moment
  • Va dépendre des solution d'hébergement
Implémentation
  • Origine : https://realpython.com/blog/python/django-development-with-docker-compose-and-machine/
  • Code de la présentation : https://github.com/encolpe/dockerizing-django
  • Présentation : https://github.com/encolpe/impress.js/tree/docker-python
DockerFile 
  • FROM python:3.5-onbuild
MAINTAINER Encolpe DEGOUTE 

#ARG build_number
#ARG app_tag

ENV PYTHONUNBUFFERED 1
Docker Engine 2/2
  • Une commande pour les construire tous

docker start
docker-compose.yml et development.yml
  • Le dévelppement étend la production 
    # Start Development
    environment:
      - DEBUG=true
    volumes:
      - ./web:/usr/src/app
      - /usr/src/app/static
# End Development
Construction 
  • docker-compose -t development.yml build
docker-compose -t development.yml up -d
docker-compose -t development.yml exec web \
     /usr/local/bin/python manage.py migrate
