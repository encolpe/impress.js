Docker avec Django

Utilisation de conteneur pour le développement Python

Encolpe DEGOUTE

  • Open Source : 1996
  • Python : 2002
  • Framework/CMS : CPS, Plone, Django, Flask
  • Principales contributions : Workflows, Template de développement

Postulats
  • Le développement est le reflet de la production
  • Le développe choisi son environnement de travail
  • Le passage du développement vers la production doit être fluide
  • L'outil de construction doit laisser la plus petite empreinte possible dans le code du projet
Contraintes
  • Doit pouvoir contenir plus que du Python (Postgres, Redis, MQ, Logstash, ElasticSearch)
  • Facile à déployer, automatision des tests
  • Linux/MacOS/Windows
  • Un non développeur doit pouvoir l'installer en une ligne de code
Pourquoi Docker
  • Déjà testé : zc.buildout, lxc, VirtualBox
  • Les plus : projet très actif, intéresse les diplomés, très suivi par des gros contributeurs
  • Les moins : victime d'incompatibilité chronique, business model incertain (futur SCO ?)
